# BUG-001

## Summary

On an invalid password, the login form shows the error text "Wrong password!" instead of the agreed copy "Invalid credentials" (violates DEMO-1 AC-2).

## Severity

red

## Linked Story

DEMO-1

## Linked Failure

FAIL-001

## Linked Risk

RISK-002

## Linked Test Case

TC-002

## Steps to Reproduce

1. Open the demo app at BASE_URL.
2. Enter username `demo` and password `nope`.
3. Press Login.

## Expected Behavior

The user remains on the login form and the error message reads exactly "Invalid credentials" (DEMO-1 AC-2 / TC-002.expected_results).

## Actual Behavior

The user remains on the login form, but the error message reads "Wrong password!" — the wrong copy. A high-severity risk (RISK-002) is therefore not met: the agreed support-facing message is not shown.

- BASE_URL: the local demo server (ephemeral port; offline)
- Browser / runtime: Chromium via Playwright
- run_id: DEMO-RUN
