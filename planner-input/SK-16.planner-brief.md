# Planner Brief — SK-16: As a user I want to complete checkout successfully

## Story summary

On Saucedemo, a logged-in user with an item in the cart completes checkout:
enters first name, last name, and postal code; the overview page shows the
correct item(s); and clicking Finish shows a confirmation message. Multi-step
E2E journey (cart → information → overview → complete); front-end-only app, no
backend order API.

## Acceptance criteria

0. user can enter first name, last name, and postal code
1. overview page shows correct items
2. finish displays confirmation message

## Risks (anchors)

- **RISK-001** (high): checkout cannot be completed at all. Related ACs: [0, 2].
- **RISK-002** (high): the overview shows the wrong items. Related AC: [1].
- **RISK-003** (medium): the information step accepts a missing required field.
  Related AC: [0].

## In-scope scenarios for the Playwright Planner

- **Complete checkout** (TC-001, RISK-001): add a known item, log in, cart →
  checkout → fill first/last/postal → continue → overview → Finish → assert the
  confirmation message and the complete URL.
- **Overview shows correct item** (TC-002, RISK-002): with a known item added,
  reach the overview and assert it lists exactly that product by name.
- **Missing-field validation** (TC-003, RISK-003): on the information step,
  leave postal code empty → Continue → assert an error and that the overview is
  not reached.

Shared setup for all three: log in as standard_user / secret_sauce and add a
known product to the cart.

## Out-of-scope for the Planner

- Login itself (SK-10) and product sorting (SK-13) beyond setup.
- Price/tax/total math on the overview (AC only mentions "correct items").
- Payment processing (Saucedemo has none).
- Cart add/remove behaviour as a feature (SK-14/SK-15) beyond setup.
- Visual styling.
