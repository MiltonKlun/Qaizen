# SK-16 — Complete checkout successfully

## Application Overview

SK-16 (Jira, Mode B): a logged-in Saucedemo user with an item in the cart completes checkout (information -> overview -> complete). E2E-only multi-step journey; front-end-only app, no order API. Locators + behaviour verified live via the playwright-test MCP on 2026-06-04 (logged in, added Sauce Labs Backpack, walked the full flow): login [data-test=username/password/login-button]; add-to-cart [data-test=add-to-cart-sauce-labs-backpack]; cart link [data-test=shopping-cart-link]; Checkout [data-test=checkout]; information step (/checkout-step-one.html) [data-test=firstName/lastName/postalCode], Continue [data-test=continue]; error [data-test=error] (empty postal -> 'Error: Postal Code is required', stays on step one); overview (/checkout-step-two.html) item names [data-test=inventory-item-name] (showed exactly 'Sauce Labs Backpack'), Finish [data-test=finish]; complete (/checkout-complete.html) header [data-test=complete-header] = 'Thank you for your order!'. Setup for each test: log in as standard_user/secret_sauce and add the backpack to the cart.

## Test Scenarios

### 1. SK-16 checkout

**Seed:** `tests/seed.spec.ts`

**File:** `tests/SK-16.spec.ts`

**Steps:**

**File:** `tests/SK-16.spec.ts`
