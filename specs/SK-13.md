# SK-13 — Sort products by name and price

## Application Overview

SK-13 (Jira, Mode B): on the Saucedemo inventory page, the product sort dropdown orders products by name (A-Z / Z-A) and price (low-high / high-low), and the chosen option stays visible as the active selection. E2E-only (Saucedemo sorts client-side; no backend sort API). Locators verified live via the playwright-test MCP on 2026-06-04: the sort control is a <select> [data-test=\"product-sort-container\"] with option values az/za/lohi/hilo; product names are [data-test=\"inventory-item-name\"]; prices are [data-test=\"inventory-item-price\"] in '$NN.NN' format; the active selection text renders in [data-test=\"active-option\"]. Verified that selecting 'lohi' reorders prices ascending (7.99..49.99) and the control shows 'Price (low to high)'. Setup: log in as standard_user / secret_sauce to reach the inventory page.

## Test Scenarios

### 1. SK-13 product sorting

**Seed:** `tests/seed.spec.ts`

#### 1.1. sort Name (A to Z) orders names ascending (TC-001)

**File:** `tests/SK-13.spec.ts`

**Steps:**

   - expect: The [data-test="inventory-item-name"] values are in ascending alphabetical order.

#### 1.2. sort Name (Z to A) orders names descending (TC-002)

**File:** `tests/SK-13.spec.ts`

**Steps:**

   - expect: The [data-test="inventory-item-name"] values are in descending alphabetical order.

#### 1.3. sort Price (low to high) orders prices ascending numerically (TC-003)

**File:** `tests/SK-13.spec.ts`

**Steps:**

   - expect: Each parsed price is >= the previous one (ascending numeric order, not string order).

#### 1.4. sort Price (high to low) orders prices descending numerically (TC-004)

**File:** `tests/SK-13.spec.ts`

**Steps:**

1. On the inventory page, select option 'hilo' and read prices, parsing each as a number.
   - expect: Each parsed price is <= the previous one (descending numeric order).

#### 1.5. selected sort option remains visible as the active selection (TC-005)

**File:** `tests/SK-13.spec.ts`

**Steps:**
