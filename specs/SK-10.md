# SK-10 — Valid user login (happy path)

## Application Overview

SK-10 (Jira, Mode B): a standard user logs into Saucedemo with valid credentials and lands on the inventory page, which shows the product list and no error message. Happy-path only; negative cases are out of scope (SK-11/SK-12). Locators verified live via the playwright-test MCP on 2026-06-04: login uses the stable test hooks [data-test="username"], [data-test="password"], [data-test="login-button"]; a successful login navigates to /inventory.html and renders product items; no error container [data-test="error"] is present on success.

## Test Scenarios

### 1. SK-10 valid login

**Seed:** `tests/seed.spec.ts`

#### 1.1. valid user logs in and reaches the inventory page (TC-001)

**File:** `tests/SK-10.spec.ts`

**Steps:**

1. Navigate to the Saucedemo login page (BASE_URL).
   - expect: The username, password fields and the Login button are visible.

2. Fill the username field with 'standard_user' using [data-test="username"].
3. Fill the password field with 'secret_sauce' using [data-test="password"].
4. Click the Login button [data-test="login-button"].
   - expect: The URL changes to /inventory.html.
   - expect: The login form is no longer shown.

#### 1.2. inventory page shows the product list after login (TC-002)

**File:** `tests/SK-10.spec.ts`

**Steps:**

1. After logging in as standard_user, observe the inventory page.
   - expect: At least one inventory item ([data-test="inventory-item"]) is visible.
   - expect: The 'Products' page title is visible.

#### 1.3. no error message is shown on a successful login (TC-003)

**File:** `tests/SK-10.spec.ts`

**Steps:**

1. Log in as standard_user with valid credentials and inspect the page for an error container.
   - expect: No error message element ([data-test="error"]) is present on the inventory page.
